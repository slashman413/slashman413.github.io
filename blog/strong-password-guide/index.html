What makes a password strong?

Three things, in order of importance:

  • Length — the single biggest factor. Each extra character multiplies the number of possible combinations.
  • Randomness — it must not be guessable. Real words, names, dates and keyboard patterns ("qwerty", "123456") are tried first by attackers.
  • Uniqueness — a different password for every site, so one breach can't unlock the rest.

Why length beats complexity

Password strength is measured in entropy (bits of unpredictability). Adding length increases entropy far faster than swapping an "a" for "@". A 16-character random password is astronomically harder to brute-force than an 8-character one, even if the short one has more symbols.

Common mistakes to avoid

  1. Reusing passwords across sites — the #1 cause of cascading account takeovers.
  2. Predictable substitutions — "P@ssw0rd!" is on every cracking list.
  3. Personal info — birthdays, pet names and phone numbers are easy to find.
  4. Never changing breached passwords — check Have I Been Pwned and rotate anything exposed.

Passphrases: strong and memorable

If you need to remember a password (like your master password), a passphrase of four or more random words — e.g. "violet-harbor-cactus-engine" — is both long and easy to recall. For everything else, let a generator make random strings and store them in a password manager.

The simple system that works

1. Use a password manager to store logins. 2. Let it (or our generator) create a unique 16+ character password per site. 3. Protect the manager with a long passphrase. 4. Turn on two-factor authentication wherever possible. You only ever memorize one passphrase.

FAQ

How long should a password be? 16 characters or more for important accounts; longer is always stronger.

Do I need to change passwords regularly? Only change them if a service is breached or you suspect exposure; forced frequent changes tend to make people pick weaker ones.

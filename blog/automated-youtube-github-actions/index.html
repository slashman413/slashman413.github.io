用 GitHub Actions 打造全自動 YouTube 頻道（免費、免伺服器）

更新於 2026 年 7 月 · 約 6 分鐘

「faceless（無人露臉）頻道」聽起來很夢幻，但多數教學不是要你買貴桑桑的工具，就是還要租伺服器。其實整條產線可以完全免費、不用自己的機器——全部跑在 GitHub Actions 的免費額度上，每天自動產一支影片並上傳。這篇拆給你看完整架構。

整條產線長這樣

1. 找題材　爬 iThome / TechOrange / Inside 等 RSS 或網頁
2. 寫腳本　用 LLM（如 DeepSeek 免費額度）產出「主持＋專家」雙人對話
3. 配音　Edge TTS（微軟免費語音）分別合成兩個角色再合併
4. 做畫面　Pillow 產投影片 + FFmpeg 加 Ken Burns 慢速縮放
5. 合成　FFmpeg 把音軌＋畫面＋背景音樂合成 720p 影片
6. 上傳　YouTube Data API v3 自動發布
7. 排程　GitHub Actions cron 每天自動跑，零維運

🔧 順手的免費工具：Cron 排程產生器（產生／解讀 GitHub Actions 的排程字串）、YouTube 縮圖下載器（抓任何影片的縮圖來做參考）。

幾個實作眉角

  • YouTube OAuth：授權一次拿到 refresh token，存進 GitHub Secrets，之後就能無人上傳。
  • 雙人語音：兩個不同的 Edge TTS 聲線分開生成、再用 FFmpeg 依對話順序串接，聽起來才像對談。
  • 字幕去標點：自動上字幕時把標點拿掉，畫面更乾淨。
  • 版權：新聞「事實」可用，但別照抄原文；用 AI 改寫成自己的口語敘述，音樂與素材要用可商用 / CC0 來源。
  • 免費額度：把頻率控制在每天 1 支，通常都落在各家免費範圍內。
